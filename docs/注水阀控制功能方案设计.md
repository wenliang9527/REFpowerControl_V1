# 注水阀控制功能方案设计文档

**版本**: V1.0  
**日期**: 2026-05-08  
**状态**: 待确认

---

## 一、功能概述

基于 SER_IN_1 信号状态，自动控制注水阀 SW_VALVE_1 的开启与关闭。系统分为**上电初始化阶段**和**系统运行阶段**，两阶段对 SER_IN_1 高电平的持续时间阈值不同。

### 1.1 核心需求回顾

| 阶段 | 触发条件 | 动作 |
|------|---------|------|
| 上电初始化 | SER_IN_1 持续高电平 ≥ 1秒 | 打开注水阀 |
| 上电初始化 | SER_IN_1 持续低电平 ≥ 5秒 | 关闭注水阀 |
| 系统运行 | SER_IN_1 持续高电平 ≥ 10秒 | 打开注水阀 |
| 系统运行 | SER_IN_1 持续低电平 ≥ 5秒 | 关闭注水阀 |

### 1.2 信号极性说明

| 信号 | 方向 | 有效电平 | 物理含义 |
|------|------|---------|---------|
| SER_IN_1 (PD12) | 输入 | 高电平 = 无液/需注水 | 液位传感器检测信号 |
| SW_VALVE_1 (PD15) | 输出 | **高电平 = 打开**，低电平 = 关闭 | 注水阀控制 |

> **极性注意**：SW_VALVE_1 的极性与 `device_control.c` 中其他设备（低电平有效）**相反**，因此注水阀控制模块将**直接操作 GPIO**，不使用 `control_device()` 接口。

## 二、信号检测实现方式

### 2.1 检测机制：TMR4 定时器中断驱动

采用与现有 `fluid_flow_interlock.c` 一致的**定时器中断 + 计数器消抖**方案：

- **时钟源**：TMR4 溢出中断，周期 **1ms**（APB1 定时器时钟 120MHz / 24 / 10000）
- **调用链**：`TMR4_GLOBAL_IRQHandler()` → `SensorInput_Scan()` → `WaterValve_Tmr4Tick()`
- **检测方式**：每次中断读取 SER_IN_1 的 GPIO 电平，累加持续时间计数器

### 2.2 选择理由

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| TMR4 中断驱动 | 与现有架构一致、时间精度高（1ms）、中断中完成无需主循环参与 | 占用少量中断执行时间 | **选用** |
| 主循环轮询 | 实现简单 | 时间精度差（主循环周期不确定）、与现有架构不一致 | 不选用 |
| 独立硬件定时器 | 精度最高 | 资源浪费、架构不统一 | 不选用 |

### 2.3 中断执行时间估算

新增 `WaterValve_Tmr4Tick()` 函数预计执行时间 < 1μs（仅计数器加减和比较），不会对 TMR4 中断的实时性造成影响。当前 TMR4 中断中已有 `TouchKey_Scan()` + `SensorInput_Scan()` + `FluidInterlock_Tmr4Tick()`，总执行时间约 5~10μs，远小于 1ms 周期。

## 三、信号状态判断逻辑与去抖机制

### 3.1 去抖策略

采用**连续计数消抖法**，与 `fluid_flow_interlock.c` 中的模式一致：

- 每次中断读取 SER_IN_1 原始 GPIO 电平
- 若当前电平与目标电平一致，对应计数器递增；不一致则**清零**
- 计数器达到阈值时确认状态变化

### 3.2 去抖阈值定义

```c
#define VALVE_INIT_HIGH_THRESHOLD_MS    1000u   /* 上电初始化: 高电平持续1秒 */
#define VALVE_RUN_HIGH_THRESHOLD_MS     10000u  /* 系统运行: 高电平持续10秒 */
#define VALVE_LOW_THRESHOLD_MS          5000u   /* 两阶段共用: 低电平持续5秒 */
```

### 3.3 去抖逻辑伪代码

```
每次 TMR4 中断 (1ms):
    读取 SER_IN_1 GPIO 电平 → current_level

    if 当前状态为"等待高电平":
        if current_level == HIGH:
            high_counter++
            low_counter = 0
            if high_counter >= 高电平阈值(根据阶段):
                → 执行打开阀门
                → 切换到"等待低电平"状态
        else:
            high_counter = 0        // 高电平中断，计数器清零

    if 当前状态为"等待低电平"（阀门已打开）:
        if current_level == LOW:
            low_counter++
            high_counter = 0
            if low_counter >= VALVE_LOW_THRESHOLD_MS:
                → 执行关闭阀门
                → 切换到"等待高电平"状态（运行阶段）
        else:
            low_counter = 0        // 低电平中断，计数器清零
```

### 3.4 去抖特性分析

- **抗毛刺**：任何短于阈值的脉冲都会被忽略（计数器清零重新计时）
- **无延迟响应**：一旦达到阈值立即执行，不存在额外延迟
- **状态保护**：阀门打开期间只监测低电平，阀门关闭期间只监测高电平，避免误触发

## 四、状态机设计

### 4.1 状态定义

```c
typedef enum {
    VALVE_STATE_WAIT_HIGH_INIT = 0,  /* 上电初始化: 等待SER_IN_1高电平≥1s */
    VALVE_STATE_WAIT_LOW_OPEN,       /* 阀门已打开: 等待SER_IN_1低电平≥5s */
    VALVE_STATE_WAIT_HIGH_RUN        /* 系统运行: 等待SER_IN_1高电平≥10s */
} valve_state_t;
```

### 4.2 状态转换图

```
                    ┌─────────────────────────┐
                    │    上电后初始状态          │
                    │  VALVE_STATE_WAIT_HIGH   │
                    │       _INIT              │
                    │                         │
                    │  监测: SER_IN_1 高电平    │
                    │  阈值: 1000ms (1秒)      │
                    └───────────┬─────────────┘
                                │
                    SER_IN_1 HIGH ≥ 1s
                                │
                                ▼
                    ┌─────────────────────────┐
                    │   VALVE_STATE_WAIT_LOW  │
                    │       _OPEN             │
                    │                         │
                    │  动作: 打开注水阀         │
                    │  监测: SER_IN_1 低电平    │
                    │  阈值: 5000ms (5秒)      │
                    └───────────┬─────────────┘
                                │
                    SER_IN_1 LOW ≥ 5s
                                │
                                ▼
                    ┌─────────────────────────┐
                    │  VALVE_STATE_WAIT_HIGH  │
                    │       _RUN              │
                    │                         │
                    │  动作: 关闭注水阀         │
                    │  监测: SER_IN_1 高电平    │
                    │  阈值: 10000ms (10秒)    │
                    └───────────┬─────────────┘
                                │
                    SER_IN_1 HIGH ≥ 10s
                                │
                                └──────► VALVE_STATE_WAIT_LOW_OPEN
                                              (阀门再次打开)
                                │
                    SER_IN_1 LOW ≥ 5s
                                │
                                └──────► VALVE_STATE_WAIT_HIGH_RUN
                                              (阀门再次关闭, 循环)
```

### 4.3 状态转换表

| 当前状态 | 条件 | 动作 | 下一状态 |
|---------|------|------|---------|
| WAIT_HIGH_INIT | SER_IN_1 HIGH ≥ 1s | 打开阀门 | WAIT_LOW_OPEN |
| WAIT_HIGH_INIT | SER_IN_1 LOW | 计数器清零，保持 | WAIT_HIGH_INIT |
| WAIT_LOW_OPEN | SER_IN_1 LOW ≥ 5s | 关闭阀门 | WAIT_HIGH_RUN |
| WAIT_LOW_OPEN | SER_IN_1 HIGH | 计数器清零，保持 | WAIT_LOW_OPEN |
| WAIT_HIGH_RUN | SER_IN_1 HIGH ≥ 10s | 打开阀门 | WAIT_LOW_OPEN |
| WAIT_HIGH_RUN | SER_IN_1 LOW | 计数器清零，保持 | WAIT_HIGH_RUN |

### 4.4 阶段转换说明

- **上电初始化 → 系统运行**：阀门首次关闭后自动进入运行阶段，不可逆
- **运行阶段循环**：WAIT_HIGH_RUN ↔ WAIT_LOW_OPEN 交替进行
- **初始化阶段超时**：若 SER_IN_1 始终为低电平，系统将一直停留在 WAIT_HIGH_INIT 状态，阀门保持关闭（安全状态）

## 六、异常情况处理

### 6.1 信号不稳定（频繁翻转）

| 异常场景 | 处理机制 | 效果 |
|---------|---------|------|
| SER_IN_1 高电平期间出现短暂低电平毛刺 | high_counter 清零，重新计时 | 毛刺不会触发阀门打开 |
| SER_IN_1 低电平期间出现短暂高电平毛刺 | low_counter 清零，重新计时 | 毛刺不会触发阀门关闭 |
| 信号持续在高低之间快速翻转 | 计数器永远无法达到阈值 | 阀门保持当前状态不变 |

### 6.2 阀门打开期间信号异常

| 异常场景 | 处理机制 | 效果 |
|---------|---------|------|
| 阀门打开后 SER_IN_1 立即变高 | low_counter 清零，重新等5秒 | 不会立即关闭，确保注水充分 |
| 阀门打开后信号消失（浮空） | GPIO 配置为上拉输入，浮空读为高 | low_counter 不计数，阀门保持打开（安全考虑：宁可多注水） |

> **安全策略**：阀门打开状态下，倾向于保持打开（fail-safe），避免因信号异常导致注水中断。

### 6.3 突发断电

| 异常场景 | 处理机制 | 效果 |
|---------|---------|------|
| 系统断电 | GPIO 输出失效，阀门由硬件回路决定默认状态 | 需确认硬件上 SW_VALVE_1 断电后阀门是否自动关闭 |
| 系统重新上电 | `WaterValve_Init()` 将状态重置为 WAIT_HIGH_INIT，阀门初始关闭 | 从头开始初始化阶段检测 |

### 6.4 阀门长时间打开保护（可选）

| 异常场景 | 处理机制 | 效果 |
|---------|---------|------|
| 阀门打开后 SER_IN_1 始终为高（传感器故障） | 可选：增加最大开启时间保护 | 超时后强制关闭阀门，防止溢水 |

> **建议**：增加最大开启时间保护（如 120 秒），阀门打开超过此时间后强制关闭并进入故障状态。此功能可作为可选配置，通过宏开关控制。

### 6.5 与现有 FluidInterlock 模块的兼容性

现有 `fluid_flow_interlock.c` 也读取 SER_IN_1 做液位检测，两者**互不干扰**：

- FluidInterlock 使用**周期性检测**（30s/60s 间隔），结果影响水泵和制冷允许
- WaterValve 使用**持续监测**，结果仅控制注水阀
- 两者在 TMR4 中断中独立运行，共享 GPIO 读取但状态机完全独立

### 8.2 对外接口

```c
/**
 * @brief  注水阀控制模块初始化
 * @note   上电后调用一次, 初始化状态机为 WAIT_HIGH_INIT,

void WaterValve_Tmr4Tick(void);

uint8_t WaterValve_IsOpen(void);

uint8_t WaterValve_GetFault(void);

/**
 * @brief  清除注水阀故障标志并恢复运行阶段检测
```

```c
void TMR4_GLOBAL_IRQHandler(void)
```

## 十一、设计总结

| 设计要素 | 方案选择 |
|---------|---------|
| 检测方式 | TMR4 定时器中断驱动（1ms 周期），与现有架构一致 |
| 去抖机制 | 连续计数消抖法，计数器清零重计 |
| 状态机 | 3 状态枚举 + if-else 分支，与 touch_key.c 风格一致 |
| 阀门控制 | 直接操作 GPIO（高电平有效），不使用 control_device() |
| 异常处理 | 毛刺免疫、断电重启恢复、可选超时保护 |
| 模块独立性 | 与 FluidInterlock 互不干扰，共享 GPIO 读取但状态机独立 |
